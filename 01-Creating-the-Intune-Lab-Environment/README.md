# Project Overview

This repository documents my hands-on Microsoft Intune Administration laboratory built within a Microsoft cloud environment.

The project demonstrates practical endpoint management skills by configuring Microsoft Intune from the ground up, enrolling real Android devices, deploying applications, configuring security policies, monitoring device compliance, and troubleshooting common administrative scenarios.

Rather than simply describing Microsoft Intune features, this portfolio documents the complete administrative workflow used by IT Support Engineers, Microsoft 365 Administrators, Endpoint Administrators and Systems Administrators in real-world enterprise environments.

This repository is the third project within the TechLab Microsoft Cloud Administration Portfolio and builds directly upon the previous Microsoft 365 and Microsoft Entra ID repositories.

---

The objectives of this project are to:

# Repository Learning Path

This repository follows the typical lifecycle of Microsoft Intune administration within an enterprise environment.

The chapters have been organised in the same order that an Endpoint Administrator would normally perform administrative tasks:

1. Create and configure the Microsoft Intune environment.
2. Become familiar with the Microsoft Intune Admin Center.
3. Prepare users and administrative permissions.
4. Enrol managed Android devices.
5. Configure compliance policies.
6. Deploy configuration profiles.
7. Deploy business applications.
8. Manage enrolled devices.
9. Secure managed endpoints.
10. Monitor devices and administrative reports.
11. Troubleshoot common support scenarios.

Each chapter combines theoretical concepts with practical hands-on exercises performed within a real Microsoft cloud laboratory.
